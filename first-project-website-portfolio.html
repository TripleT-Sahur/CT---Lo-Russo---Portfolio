The Project Portfolio Website

While creating this website, I intended it to be a reflection of my computer technology skills at the start of the unit. I experimented with colours, animation, and presentation of content while keeping the flexibility of adding more projects/achievements in future.

With the use of flexbox, the website became versatile and accessible on varying monitor sizes, including mobile devices. If the window is resized, then the majority of items will be resized along with it, some will even have their dimensions and ratios changed to fit the screen. This allows people to be able to view the website on any device. In order to add a feeling of depth and progression to a viewer making their way through the website, I made the background colour a gradient, so it would progressively get darker the further someone scrolled down.

To add the side-scrolling mechanic in the homepage without using JavaScript, I turned the overflow method to scrolling and intentionally made the container wider than the page. I did encounter an issue with the object clipping out one of the sides, but that was solved with removing the Justify Content - Space Around line. I also managed to hide the scrollbar on the x-axis, which was causing issues with blocking content. By using the <>:hover {transition:<>} code, I was able to add transitions to specific objects when hovering over them. This ranged from slight alterations to colour and size, with some actually turning an image into text!

The main method of saving and loading the website was git, specifically GitHub, which allows you to share, publish, save, compare, and acts as an essential platform for computer technologists and web developers alike. Using ‘commits’ and ‘branches’, I was able to capture specific states of the code, which also allowed me to go over additions and subtractions in chronological order.
